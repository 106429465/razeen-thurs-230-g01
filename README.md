## Commit syntax
* `feat` new feature.
* `fix` bug fix.
* `docs` documentation or comments.
* `style` formatting or CSS style changes, no logic change.
* `refactor` code change that improves structure, no behaviour change.
* `perf` performance improvement.
* `test` add or update tests.
* `build` build tools, dependencies.
* `ci` pipelines and scripts.
* `chore` small tasks like renames or clean up.
* `revert` undo a past commit.

**commit_type(scope): commit_details**

_e.g._ feat(form.html): add reset button to form
